A modal for a routine yes/no confirmation. It reuses the headless dialog
(`role="dialog"`) and the shared modal adapter: portal, focus trap, scroll lock.
The trigger is the default slot; `onConfirm` runs when the confirm button is
pressed. Focus lands on the safe choice.

The example reuses the same wording as [Alert Dialog](/invisible-ui/components/overlays/alert-dialog/)
so the family reads consistently — the only difference is the role and urgency
(`role="dialog"` here vs `role="alertdialog"` for the interrupting variant).

### Wording

Buttons name outcomes, not gestures. The classic trap: *"Do you want to cancel
the order?"* with **Cancel** / **OK** buttons — "Cancel" reads both as "cancel
the order" and "close this dialog", and a screen-reader user hears the bare
label with no layout around it to disambiguate. Name what each choice does
instead: **Cancel order** / **Keep order** (or, in the example below,
**Delete file** / **Keep file**). Both labels are overridable; the defaults are
the conventional "Confirm"/"Cancel", and the examples here always name the
outcome.

### When to use which dialog

- **[Dialog](/invisible-ui/components/overlays/dialog/)** — generic modal for any content.
- **Confirm Dialog** — a routine yes/no confirmation (`role="dialog"`).
- **[Alert Dialog](/invisible-ui/components/overlays/alert-dialog/)** — an *urgent* or *destructive* action
  that must interrupt and be acknowledged (`role="alertdialog"`, optional
  type-to-confirm).
- **[Prompt Dialog](/invisible-ui/components/overlays/prompt-dialog/)** — asks for a single value.

### Accessibility

- `role="dialog"`, `aria-modal`, named by its `title` and described by the
  optional message. Escape and a backdrop press cancel (set
  `closeOnOutsideClick={false}` to require an explicit choice).
- Focus is trapped in the panel and starts on the Cancel button; it returns to
  the trigger on close.

### Example

```svelte
<ConfirmDialog
  title="Delete file"
  description="Do you want to delete “report-q3.pdf”? The file will be permanently deleted."
  confirmLabel="Delete file"
  cancelLabel="Keep file"
  confirmVariant="danger"
  onConfirm={() => remove()}
>
  Delete file
</ConfirmDialog>
```

Themeable via `--ds-dialog-*`.
