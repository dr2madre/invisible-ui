A modal window (WAI-ARIA dialog pattern): a trigger opens a `role="dialog"` panel
with `aria-modal="true"`, built on the **native `<dialog>` element** and
`showModal()`: top-layer rendering, an inert background (a real focus trap,
enforced by the browser for keyboard *and* assistive tech), a stylable
`::backdrop`, plus body scroll lock.

This is the generic modal the rest of the family builds on. The three presets
are the accessible, stylable equivalents of the platform's *simple dialogs*
(`alert()`, `confirm()`, `prompt()`): same headless machinery, same tokens,
differing only in job.

### When to use which dialog

- **Dialog** — generic modal for any content; the head of the family.
- **[Alert Dialog](/invisible-ui/components/feedback/dialog/alert-dialog/)** — an acknowledgement: a message with
  nothing to cancel (one button).
- **[Confirm Dialog](/invisible-ui/components/feedback/dialog/confirm-dialog/)** — verify or accept before proceeding:
  cancel stops, confirm proceeds (`urgent` to interrupt).
- **[Prompt Dialog](/invisible-ui/components/feedback/dialog/prompt-dialog/)** — asks for a single value
  (optional, `required`, or gated by `confirmValue`; `urgent` to interrupt).
- **[Search Dialog](/invisible-ui/components/feedback/dialog/search-dialog/)** — a quick search: search by keyword and
  pick from a list.
- **[Sheet Dialog](/invisible-ui/components/feedback/dialog/sheet-dialog/)** — a task in an edge-anchored panel
  (`side` variant, optional drag-to-dismiss).

### While a dialog is open, the page waits

A modal interrupts by design: `showModal()` makes everything outside it
**inert** — non-interactive and silenced for assistive technology too. That
includes floating [Notifications](/invisible-ui/components/feedback/notification/):
a toast raised while a dialog is open renders underneath it and is not
announced. This is coherent, not a limitation — background information waits;
anything about the task in progress belongs *inside* the dialog.

**Feedback inside a dialog** — for a non-blocking message tied to the task
(a failed save, a lost connection), render an
[Inline Notification](/invisible-ui/components/feedback/inline-notification/)
in the dialog body. Inside the open dialog it is outside the inert zone, so its
live region announces normally:

```svelte
<Dialog title="Edit profile">
  {#if saveError}
    <InlineNotification status="danger" title="Save failed" role="alert" />
  {/if}
  <form>…</form>
</Dialog>
```

**Dialogs stack — one level, one job.** The top layer is a stack: opening a
dialog from inside a dialog puts it on top and makes the outer one inert too.
The one legitimate case is the **confirm-on-close**: a Confirm Dialog guarding
an edit dialog with unsaved changes. **Escape pops one level** (the confirm
closes, the editor survives, focus returns inside it); scroll stays locked
until the last dialog closes. If you need more than one level of stacking, the
task wants a different shape, not a deeper stack.

```svelte
<Dialog title="Edit profile">
  <form>…</form>
  <ConfirmDialog
    title="Discard changes?"
    description="Your edits will be lost."
    confirmLabel="Discard changes"
    cancelLabel="Keep editing"
  >
    Discard
  </ConfirmDialog>
</Dialog>
```

### Accessibility

- Trigger carries `aria-haspopup="dialog"` / `aria-expanded` and, while open,
  `aria-controls`. The panel is `role="dialog"` with `aria-modal="true"`, named
  by its title (`aria-labelledby`) and optionally described (`aria-describedby`).
- Modal semantics: focus moves into the panel on open, is **trapped** (Tab
  wraps), body scroll is **locked**; **Escape**, the close button, or a backdrop
  press close it and return focus to the trigger.
- Portalled to `<body>` to escape ancestor `overflow`/stacking contexts.

### Example

```svelte
<Dialog title="Edit profile" description="Make changes to your profile here.">
  <span slot="trigger">Edit profile</span>
  <form>…</form>
</Dialog>
```

Themeable via `--ds-dialog-*` (the backdrop via `::backdrop`). The shared
modal adapter — native `<dialog>` + `showModal()` and scroll lock — is reused
across the whole dialog family.
