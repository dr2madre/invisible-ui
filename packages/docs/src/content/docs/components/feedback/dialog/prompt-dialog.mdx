A modal that asks the user for a single value ‚Äî the accessible equivalent of
`window.prompt()`, one of the platform's *simple dialogs*: everything
[Confirm Dialog](/invisible-ui/components/feedback/dialog/confirm-dialog/) has, plus an
input. It reuses the headless dialog and the shared modal adapter (native
`<dialog>` + `showModal()`: top layer, inert background, scroll lock). The input is focused on open and Enter confirms.
`onConfirm(value)` runs with the entered text.

The value is **optional by default** ‚Äî context decides: set `required` to keep
the confirm button disabled while blank, or **`confirmValue`** for a
type-to-confirm gate, where the button stays disabled until the input matches
exactly ‚Äî e.g. typing a file name to confirm its deletion (the second demo
above). Pair the gate with `confirmVariant="danger"` and **`urgent`** for
destructive actions: `urgent` switches the panel to `role="alertdialog"`
(announced immediately by screen readers) and changes nothing else.

### When to use which dialog

- **[Dialog](/invisible-ui/components/feedback/dialog/)** ‚Äî generic modal for any content; the head of the family.
- **[Alert Dialog](/invisible-ui/components/feedback/dialog/alert-dialog/)** ‚Äî an acknowledgement: a message with
  nothing to cancel (one button).
- **[Confirm Dialog](/invisible-ui/components/feedback/dialog/confirm-dialog/)** ‚Äî verify or accept before proceeding:
  cancel stops, confirm proceeds (`urgent` to interrupt).
- **Prompt Dialog** ‚Äî asks for a single value
  (optional, `required`, or gated by `confirmValue`; `urgent` to interrupt).
- **[Search Dialog](/invisible-ui/components/feedback/dialog/search-dialog/)** ‚Äî a quick search: search by keyword and
  pick from a list.
- **[Sheet Dialog](/invisible-ui/components/feedback/dialog/sheet-dialog/)** ‚Äî a task in an edge-anchored panel
  (`side` variant, optional drag-to-dismiss).

### Accessibility

- `role="dialog"`, `aria-modal`, named by its `title`; the input is labelled by
  `label` and receives focus on open. Escape and a backdrop press cancel.
- Focus is trapped in the panel and returns to the trigger on close.

### Example

```svelte
<!-- Type the file name to enable the destructive Delete. -->
<PromptDialog
  urgent
  title="Delete file"
  description="Do you want to delete ‚Äúreport-q3.pdf‚Äù? The file will be permanently deleted."
  label="Type the file name to confirm"
  placeholder="report-q3.pdf"
  confirmValue="report-q3.pdf"
  confirmLabel="Delete file"
  cancelLabel="Keep file"
  confirmVariant="danger"
  onConfirm={() => remove()}
>
  Delete file
</PromptDialog>
```

Themeable via `--ds-dialog-*`.
