A modal to verify or accept before proceeding — the accessible equivalent of
`window.confirm()`, one of the platform's *simple dialogs*. Cancel stops the
process, confirm proceeds. It reuses the headless dialog and the shared modal
adapter (native `<dialog>` + `showModal()`: top layer, inert background,
scroll lock). The trigger is the default slot;
`onConfirm` runs when the confirm button is pressed. Focus lands on the safe
choice.

Set **`urgent`** when the choice must interrupt: the panel switches to
`role="alertdialog"` (screen readers announce it immediately) and nothing else
changes — urgency is a modifier, not a different component (the demo above
shows both).

It also works **stacked on another dialog** — the confirm-on-close case: a
Confirm Dialog guarding an edit dialog with unsaved changes. The top layer
stacks it above, Escape closes only the confirm, and focus returns into the
outer dialog (see
[the family overview](/invisible-ui/components/feedback/dialog/#while-a-dialog-is-open-the-page-waits)).

### Wording

Buttons name outcomes, not gestures. The classic trap: *"Do you want to cancel
the order?"* with **Cancel** / **OK** buttons — "Cancel" reads both as "cancel
the order" and "close this dialog", and a screen-reader user hears the bare
label with no layout around it to disambiguate. Name what each choice does
instead: **Cancel order** / **Keep order** (or, in the example below,
**Delete file** / **Keep file**). Both labels are overridable; the defaults are
the conventional "Confirm"/"Cancel", and the examples here always name the
outcome.

### When to use which dialog

- **[Dialog](/invisible-ui/components/feedback/dialog/)** — generic modal for any content; the head of the family.
- **[Alert Dialog](/invisible-ui/components/feedback/dialog/alert-dialog/)** — an acknowledgement: a message with
  nothing to cancel (one button).
- **Confirm Dialog** — verify or accept before proceeding:
  cancel stops, confirm proceeds (`urgent` to interrupt).
- **[Prompt Dialog](/invisible-ui/components/feedback/dialog/prompt-dialog/)** — asks for a single value
  (optional, `required`, or gated by `confirmValue`; `urgent` to interrupt).
- **[Search Dialog](/invisible-ui/components/feedback/dialog/search-dialog/)** — a quick search: search by keyword and
  pick from a list.
- **[Sheet Dialog](/invisible-ui/components/feedback/dialog/sheet-dialog/)** — a task in an edge-anchored panel
  (`side` variant, optional drag-to-dismiss).

### Accessibility

- `role="dialog"`, `aria-modal`, named by its `title` and described by the
  optional message. Escape and a backdrop press cancel (set
  `closeOnOutsideClick={false}` to require an explicit choice).
- Focus is trapped in the panel and starts on the Cancel button; it returns to
  the trigger on close.

### Example

```svelte
<ConfirmDialog
  title="Delete file"
  description="Do you want to delete “report-q3.pdf”? The file will be permanently deleted."
  confirmLabel="Delete file"
  cancelLabel="Keep file"
  confirmVariant="danger"
  onConfirm={() => remove()}
>
  Delete file
</ConfirmDialog>
```

Themeable via `--ds-dialog-*`.
