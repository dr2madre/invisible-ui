A loading **area**: a surface that signals a process in progress, from two
composable parts.

1. A **loading visual** — by default the **halftone dot field** (`field`): a
   faint dot lattice with a couple of soft, brighter **zones** that drift across
   it continuously (like the "image generating" dot cards), so different areas
   seem to render over time. The dots are a CSS `background` (no per-dot DOM),
   so it **tiles to any size** cheaply.
2. A label **zone**, placed with `labelPosition` (`center` default, or
   `top` / `bottom` / `left` / `right`). It shows any combination of:
   - a live `status` message ("Connecting…", "Rendering…"),
   - a `value` percentage (shown as `N%`),
   - a `detail` line — free text for **bytes or a file count** ("48 MB of 128
     MB", "3 of 8 files").

The dot field and an explicit indicator are **alternatives**, never stacked (no
loader on a loader): turn `field` off and put a different loader — e.g. a
[Loading](/invisible-ui/components/feedback/loading/) `spinner` — into the
`indicator` slot to use that as the area's loader instead.

When the process finishes, set `loading={false}`: LoadingGenerationArea renders its **default
slot** (the real content) in place of the placeholder, with a gentle fade.

Unlike the inline `Loading` variants, LoadingGenerationArea **covers a surface**; use `Loading`
for an inline indicator next to text or in a button.

### Import

```svelte
<script>
  import LoadingGenerationArea from "@design-system/svelte/LoadingGenerationArea.svelte";
</script>
```

### Usage

Give it a sized, positioned container (or drop it in as a filling layer). Add a
label zone (position + status / percentage / detail), and optionally an explicit
indicator through the slot:

```svelte
<div style="position: relative; aspect-ratio: 16 / 10;">
  <!-- Percentage + file count in a bottom label zone -->
  <LoadingGenerationArea labelPosition="bottom" status="Uploading" value={pct} detail="3 of 8 files" />
</div>

<div style="position: relative; aspect-ratio: 16 / 10;">
  <!-- A plain area (no dot field) with an explicit spinner via the slot -->
  <LoadingGenerationArea field={false} label="Loading">
    <Loading slot="indicator" variant="spinner" decorative />
  </LoadingGenerationArea>
</div>
```

When the data arrives, flip `loading` and the default slot's content replaces the
placeholder (with a gentle fade):

```svelte
<LoadingGenerationArea loading={pending} status="Generating image">
  <!-- the generated result (an image, a mock, any content) — shown once `pending` is false -->
  <img src={url} alt="Generated image" style="width: 100%; height: 100%; object-fit: cover;" />
</LoadingGenerationArea>
```

### Accessibility

- A polite `role="status"` with an accessible name (`label`; when omitted it
  comes from the i18n catalog, so `LocaleProvider` localizes it) — it announces
  the loading state like the other indicators.
- When `status` is set it drives the announcement (the region is `aria-atomic`,
  so each new message is read in full); the `value` percentage and `detail` are
  visible but `aria-hidden`, so a fast-ticking value isn't re-announced.
- Set `decorative` when the surrounding region already announces that it is busy
  (e.g. an `aria-busy` container) — the field is then hidden from assistive tech.
- The drift respects `prefers-reduced-motion`: the animation stops and the glows
  hold still (a gentle static brightening).

### Theming

Colors follow `currentColor` (set `color` on the container). Density and speed
are themeable:

| Token | Purpose | Default |
| --- | --- | --- |
| `--ds-loading-generation-area-dot` | Resting (grey) dot radius | `1.4px` |
| `--ds-loading-generation-area-dot-lit` | Lit dot radius (dots grow to this in a zone) | `3px` |
| `--ds-loading-generation-area-gap` | Distance between dots | `1.1rem` |
| `--ds-loading-generation-area-duration` | Drift cycle length | `7s` |
| `--ds-loading-generation-area-min-height` | Fallback min height | `6rem` |
