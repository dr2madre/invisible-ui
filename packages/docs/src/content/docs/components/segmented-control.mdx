The radio-group semantics laid out as a bar of segments. Beyond the default
text bar it supports `iconOnly` (a compact toolbar), `stacked` (icon over label —
a mobile **bottom bar**), and `orientation="vertical"` (a **sidebar**); combine
them for an icon-only or compact sidebar. The demo above shows each variant,
including a desktop/tablet sidebar with a logo and `Separator` dividers.

### Import

```svelte
<script>
  import SegmentedControl from "@design-system/svelte/SegmentedControl.svelte";
</script>
```

### Props

<PropsTable component="segmented-control" />

### Accessibility

- Built on native `<input type="radio">` segments: the browser handles roving
  tabindex, arrow navigation and form participation.
- The control needs an accessible name via `label`.

### Example

```svelte
<SegmentedControl
  label="View"
  value="list"
  items={[
    { value: "list", label: "List" },
    { value: "grid", label: "Grid" },
  ]}
  onValueChange={(v) => (view = v)}
/>
```

Themeable via `--ds-segment-*`.
